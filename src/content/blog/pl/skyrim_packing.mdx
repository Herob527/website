## Wstęp

Witam w drugim wpisie, w którym pokrywam temat dubbingowania modyfikacji do Skyrima.

Ten artykuł skupi się na rozpakowaniu i spakowaniu plików bsa poprzez `BSArch` oraz rozbijaniu i składaniu pliku `.fuz` poprzez `YakitoriAudioConverter`.

Dodatkowo powiem co nieco o strukturze archiwum bsa z głosami oraz przegląd struktury paru modyfikacji, które zawierają dubbing (tu: [Alternate Start](https://www.nexusmods.com/skyrimspecialedition/mods/182007) i [Gifts of the Outsider](https://www.nexusmods.com/skyrimspecialedition/mods/182701)).

### BSArch

#### Rozpakowywanie i pakowanie bsa

By rozpakować, należy najpierw stworzyć folder docelowy a potem dopiero uruchomić bsarch

<CodeVariant items={[
{
code:
`
mkdir output
BSArch.exe unpack ./Gifts\ Of\ The\ Outsider.bsa output
`,
lang: 'bash',
name: "Windows"
},
{
code:
`
export WINEPREFIX=$HOME/.local/share/wineprefixes/wine_dotnet
mkdir output
wine BSArch.exe unpack ./Gifts\ Of\ The\ Outsider.bsa output 2>>/dev/null
`,
lang: 'bash',
name: "Linux"
}

]} />

Pozamienialiśmy pewne pliki to czas spakować z powrotem do bsa

<CodeVariant items={[
{
code:
`

# -sse - Pakuje dla Skyrim Special Edition

BSArch.exe pack "output" "Gifts Of The Outsider.bsa" -sse
`,
lang: 'bash',
name: "Windows"
},
{
code:
`
export WINEPREFIX=$HOME/.local/share/wineprefixes/wine_dotnet

# -sse - Pakuje dla Skyrim Special Edition

wine BSArch.exe pack "output" "Gifts Of The Outsider.bsa" -sse 2>>/dev/null
`,
lang: 'bash',
name: "Linux"
}

]} />

Poza `-sse` można pakować też dla Falloutów, Obliviona, Skyrima czy Starfielda

Lista opcji ustalających format dla docelowej gry:

- `-tes3` - Format archiwum Morrowinda
- `-tes4` - Format archiwum Obliviona
- `-fo3` - Format archiwum Fallout 3
- `-fnv`- Format archiwum Fallout New Vegas
- `-tes5`- Format archiwum Skyrima LE (wersja przed SE)
- `-sse`- Format archiwum Skyrim SE/AE
- `-fo4`- Format archiwum Fallout 4
- `-sf1`- Format archiwum Starfielda

#### Struktura bsa z głosami

Archiwum bsa może zawierać w sobie dźwięki efektów specjalnych, tekstury, skrypty, pliki głosów przypisane dla danego _voice type_ i reszta potrzebne modowi pliki.

Podam parę przykładów, jak wygląda struktura ze skupieniem się na pliku audio.

Nazwy będą odpowiednio skrócone, by mieściły się zwłaszcza na mniejszych urządzeniach, bo realne będą o wiele dłuższe.
